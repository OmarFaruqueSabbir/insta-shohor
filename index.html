Javascript Questionnaires

JS works: JS is mainly a single-threaded interpreted programming language. This refers to JS executing one instruction or a part of code in a single time sequence. JS ensures dynamic typing while runtime. By nature, JS is synchronous. JS maintains order and runs code top to bottom manner. But in case of asynchronous operation, the servers await for a network request in a loop and accepts further requests when the first one is being processed.

Event Loop: As we know JS is synchronous. But in the case of an asynchronous operation, event loop comes in. It continuously checks if there's anything to execute in the call stack, otherwise, it checks the callback queue. If something is in the callback queue, the event loop will move it to the call stack - if only the call stack remains empty! So the event loop is a never-ending process whose sole purpose is to monitor and transmit methods from the event queue to the call stack.

LocalStorage vs SessionStorage: Both localStorage & sessionStorage is similiar in manner. But, difference occurs in case of accessing & storing data.
1. LocalStorage can be accessed by any browser window or tab.
2. SessionStorage can only be accesed within same tab or window.
3. LocalStorage stores data with no expiry but can be deleted manually if needed.
4. Stored data in SessionStorage gets cleared when a session ends.
